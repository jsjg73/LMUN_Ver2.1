1. 모임 시나리오
    1.1 생성 시나리오
        - 웹 메인 화면의 모임 생성 버튼이 있다.
        - 모임 생성 화면에서 모임이름, 최소 인원을 입력한다.
        - 비회원은 모임에서 사용할 이름과, 관리자 비밀번호를 함께 입력해야한다.
        - 모임과 함께 식별 정보 13자리가 생성된다. 비회원의 경우 이 식별정보가 비회원 임시계정의 ID가 된다.
        - 모임을 생성한 사용자가 그 모임의 관리자로 등록된다.
        - 모임 생성 로그 메시지 "관리자 OO님이 모임을 생성했습니다."를 모임 메인 화면의 로그메시지 창에 출력한다.

    1.2 입장 시나리오
        - 사용자가 모임에 최초로 입장할 때는 모임 내에서 사용할 이름과 출발지가 필요하다. 회원은 등록된 정보가 자동으로 사용된다.
        - 모임에 최초로 참가하는 방법은 두 가지다.
            a. 웹 메인에서 모임 식별자를 검색.
            b. URL로 직접 입장.(도메인/entrance/식별정보)
        - 식별정보를 모르면 입장할 수 없다. 단, 재입장의 경우는 참가한 모임 목록을 통해 들어갈 수 있다.
        - 비회원일 경우 입장화면 전에 로그인 혹은 가입을 제안한다.
        - 비회원 최초 입장
            - 비회원 입장 화면을 보여준다. 모임명, 현재 참석자 목록이 표시된다. 
            - 새로운 참가자 버튼을 누르면 이름과 패스워드, 출발지를 입력할 수 있는 공간이 나타난다.
            - 모임 식별 정보를 기반으로해서 새 비회원 참가자의 ID를 생성한다. 이름,패스워드 그리고 출발지를 함께 데이터베이스에 등록한다.
        - 비회원 재입장
            - 비회원 입장화면의 참석자 목록에서 선택하고 패스워드를 입력한다.
            - 잘못된 패스워드일 경우 알림을 띄우고 페이지를 유지한다.
        - 입장 로그 메시지 "OO님이 입장했습니다"를 모임 메인 화면의 로그메시지 창에 출력한다.

    1.3 참석자 출발지 변경 시나리오
        - 모임 메인 화면에서 '내 출발지 변경' 버튼을 클릭하면 출발지를 변경할 수 있다.
        - 출발지 변경 로그 메시지 "OO님이 출발지를 변경했습니다."를 모임 메인 화면의 로그메시지 창에 출력한다.

    1.4 추천 지역 검색 시나리오
        - 모임 메인 화면에서 장소 검색 버튼을 누르면 '추천 지역 검색 화면'으로 이동한다.
        - 참석자들의 출발지 정보를 바탕으로 추천 지역 목록을 보여준다. 추천 지역 목록은 외부 API를 이용한다.
        - 각 출발지로 부터 가장 공평한 위치를 구한다. 앞으로 이를 중심이라 부른다.
        - 중심은 모든 출발지 좌표의 평균으로 정한다. 중심 언제든지 변경할 수 있도록 구현해야한다.
        - 추천지역이 최대 5곳이 나올때 까지 중심에서 부터 500미터 단위로 범위를 넓혀가며 검색한다. (최대 3km)
        - 3km 까지 검색해도 추천 지역이 없을 경우, 가장 가까운 장소 1개를 검색해서 보여준다.
        - 사용자는 추천 지역중에서 하나를 제안할 수 있다. 제안하기를 누르면 "OO님이 OO장소를 제안했습니다"라는 로그 메시지가 생성된다.
        - 제안된 추천 지역의 정보는 제안자 정보와 함께 DB에 저장된다.

    1.5 제안 지역 상세 보기 시나리오
        - 제안자가 추천 지역을 탐색하거나, 다른 참석자의 제안을 확인할 때 '제안 지역 상세 보기' 화면으로 이동한다.
        - 제안 지역 상세 보기 화면은 다음 정보를 지닌다. (제안자), 장소명, 식별자, 주소, 유형, 평균 시간, 평균 거리, 각 참석자들의 경로, 거리, 시간, 주변 시설 및 가게.
        - 제안자가 아닌 다른 참석자들은 해당 제안이 맘에들 경우 동의를 표시할 수 있다.

2. 회원 시나리오
    2.1 회원 계정 시나리오
    2.2 비회원 계정 시나리오


    
    
사용자가 모임을 개설한다. 개설할 때 모임명을 등록한다. 그 사용자는 모임의 관리자로 등록된다. 관리자는 모임의 최소인원 및 모임명을 변경할 수 있다. 그리고 모임을 삭제할 수 있다. 모임을 생성하면 식별정보가 함께 생성된다. 다른 사용자들은 이 식별 정보를 이용해서 참석할 수 있다. 모임명을 통한 검색은 제공하지 않는다.

모임 메인화면에서 모임명, 참석자 명단, 로그(입장, 퇴장, 제안, 모임 정보 수정), 장소 제안 상세 내용, 주변 시설 정보 를 볼 수 있어야 한다.
 
모임에 참석한 사용자는 참석자라고 부른다. 한 모임에 참석자는 최대 ( )명이다. 2명 이상이 출발지를 입력하면 참석자들 중 누구든지 추천 장소를 검색할 수 있다. 검색 결과는 해당 지역까지 참석자들의 대략적인 이동 경로, 소요시간 및 거리 정보를 표시한다. 참석자들은 자신이 검색한 결과를 모임에 제안할 수 있다. '장소 제안'은 모임 메인 화면에서 로그 메시지로 확인할 수 있다("xx님이 xx에서 모이자고 제안합니다."). 상세 보기를 누르면 다른 사용자들도 해당 지역까지의 이동 경로, 소요시간 및 거리 정보를 볼 수 있어야한다.

모든 참석자의 동의를 얻은 제안은 약속 장소로 선택된다. 확정된 약속 장소 주변 시설을 볼 수 있다. 주변 시설은 카테고리(대형마트, 학교, 주차장, 치하철역, 은행, 문화시설, 관광명소, 음식점, 카페)를 선택할 수 있고, 목록 형태로 보여준다. 원하면 상세 정보를 확인할 수 있어야한다.

기본 서비스는 특별한 회원가입 없이도 이용할 수 있어야한다. 회원으로 등록하면 모임에 참석할 때 미리 저장한 이름과 출발지를 사용할 수 있다. 비회원은 모임에 참석할 때 참석자 이름과 출발지를 직접 입력해야한다.

참석자가 모임에서 퇴장하면 모임 메인 화면의 참석자 명단에서 제거된다. 퇴장한 참석자가 포함된 제안들은 비활성화 된다. 하지만 기존의 장소 제안과 선택은 유지된다. 퇴장한 참석자의 정보만 비활성화된다. 즉, 참석자의 퇴장으로인해 추천장소가 다시 계산되진 않는다.

회원이 탈퇴하면 모든 모임에서 퇴장한다. 회원이 관리자로 있는 모임은 삭제된다.




## 시나리오 보강이 필요한 부분

* 사용자가 모임에 입장할 때 시나리오 보강. ex) 비회원이 브라우저 종료 후 다시 입장할 때, 동일한 사용자임을 확인하는 방법.

* 사용자가 퇴장할 때 시나리오 보강.

* 퇴장한 사용자가 다시 입장할 때 시나리오 보강.
